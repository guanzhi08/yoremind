# 十一、AI 自動化功能

本章描述 LocaRemind 如何以 AI 作為中間層，將非結構化輸入（自然語言、截圖、語音）自動轉換成結構化任務，並在指派、排程、監控三個環節實現自動化，大幅降低照顧者與一般使用者的操作負擔。

---

## 11.1 核心理念

```
傳統流程：使用者手動填寫 任務名稱 / 地點 / 時間 / 備註 → 四個欄位逐一輸入

AI 流程：  使用者輸入自然語言 / 上傳截圖 / 說話
          ↓  AI 解析（LLM + OCR + STT）
          自動填入：任務名稱、地點座標、時間視窗、提醒備註
          ↓  使用者一鍵確認或微調
          任務建立完成，觸發邏輯自動生效

核心價值：將任務建立從「填表操作」轉變為「說話 / 拍照即完成」，
         降低所有使用者（特別是家人、社工）的操作摩擦。
```

---

## 11.2 任務建立自動化（Capture → Record）

### 1. 自然語言建立任務

使用者輸入一段文字描述，AI 自動解析並填入所有結構化欄位：

| 使用者輸入（自然語言） | AI 自動解析結果 |
|----------------------|----------------|
| 提醒阿公週四下午到長庚診所看診，記得帶藥袋和健保卡 | • 任務名稱：長庚診所回診<br>• 地點：長庚醫院（自動地圖定位）<br>• 時間視窗：週四 14:00–18:00<br>• 提醒備註：帶藥袋和健保卡 |
| 下週一要去全聯買牛奶跟雞蛋，記得結帳前確認有沒有折扣 | • 任務名稱：全聯採購<br>• 地點：最近的全聯（地圖搜尋）<br>• 時間：下週一（全天觸發）<br>• 清單：牛奶、雞蛋；備註：確認折扣 |

---

### 2. 截圖 / 圖片解析

使用者上傳圖片（醫囑單、取貨通知、預約確認截圖），AI 執行 OCR + 語意理解，自動填入任務欄位：

- 電商取貨簡訊截圖 → 自動建立「取貨提醒」（地點：對應超商、時效：截止日倒數提醒）
- 醫院預約確認截圖 → 自動建立「回診提醒」（醫院地點、時段、科別、看診序號）
- 藥袋照片 → 解析用藥頻率（每日幾次、飯前飯後），建立定期服藥或領藥提醒
- 包裹到貨通知截圖 → 自動建立「超商取貨提醒」並設定 7 天倒數到期警示

---

### 3. 語音輸入

社工或家人語音說明任務 → 語音轉文字（STT）→ NLU 解析 → 輸出結構化任務。特別適合在移動途中、雙手不方便操作時快速建立任務。

| 語音輸入情境 | AI 處理流程 |
|------------|------------|
| 社工在前往個案家途中語音建立任務 | STT 辨識 → 解析個案姓名 + 任務內容 → 帶入對應個案帳號 → 確認後建立 |
| 家人語音設定長者外出提醒 | STT → NLU 解析地點與時間 → 在長者帳號建立任務 → 家人確認後啟用 |

---

## 11.3 批次指派自動化（社工場景）

### 1. 自然語言批次指派

社工輸入一句話，AI 自動篩選符合條件的個案並執行批次操作：

```
社工輸入：「通知所有週三日照的個案，本週改為週四，請帶毛衣」

AI 執行：
  ① 篩選符合條件的個案（排程中有「週三日照」任務者，共 8 位）
  ② 批次更新任務時間（週三 → 週四）
  ③ 附加備註：帶毛衣
  ④ 發送確認通知給各個案的家人（FCM 推播）
  ⑤ 產生操作摘要：「已更新 8 位個案的日照排程，3 位家人已讀，5 位待確認」
```

---

### 2. 醫囑自動解析與任務生成

護理師上傳出院摘要或醫囑單圖片，AI 解析後自動生成完整任務清單，一鍵推送到長者帳號：

- 回診排程：日期 + 科別 + 地點 → 自動建立地理圍欄觸發任務
- 用藥提醒：藥名、頻率（每日幾次）、領藥地點 → 建立定期時間觸發任務
- 復健計畫：頻率 + 復健診所地點 → 建立週期性地點觸發任務
- 家人自動收到授權確認通知，一鍵啟用後任務正式生效

---

### 3. 個案狀態摘要（社工 AI 助理）

系統每日自動產生個案狀態摘要，聚焦需要追蹤的異常狀況：

```
📋 今日個案狀態摘要（2026-04-29 08:00）

• 本週 5 位個案中，3 位任務完成率 100%
• 王阿嬤：本週漏接藥局提醒 2 次（週二 14:00、週四 14:00），建議追蹤
• 陳爺爺：昨日超出安全圍欄 1 次（已返回，持續 12 分鐘），家人已知悉
• 李奶奶：本週回診任務明日到期，尚未完成，建議提前確認交通安排

需要處理的項目：2 件  ｜  可自動處理：1 件
```

## 11.4 異常偵測與預測（Remind 端）

### 1. 行為模式學習

AI 分析長者過去 30 天的出行紀錄，建立正常行為基線，作為異常偵測的比對依據：

| 行為維度 | 學習內容 |
|---------|---------|
| 出門時間 | 通常幾點出門、活動時間段（上午 / 下午） |
| 常去地點 | 高頻地點（公園、超商、社區活動中心）與到訪頻率 |
| 移動時長 | 單次外出平均時長、最長停留時間 |
| 返家模式 | 通常幾點返家、外出後的典型路線 |

---

### 2. 異常警示

偵測到偏離基線的行為時，系統自動升級通知層級：

- 長者在非常規時間（深夜 22:00 後、清晨 05:00 前）離開家 → 即時通知家人
- 停留在非常規地點（首次出現的地點）超過設定時長 → 推播狀態確認
- 預計返家時間已過（依歷史平均外出時長推算）但尚未回到安全圍欄 → 分級警示
- 任務完成率連續 3 天低於 50% → 通知社工進行關懷追蹤

---

### 3. 多點路徑最佳化

當日有多個地點任務時，AI 建議最佳拜訪順序，減少不必要的往返：

- 輸入：當日任務地點清單（藥局、超商、醫院、銀行）
- 考量因素：各任務的時間視窗、地點間距離、交通方式（步行 / 大眾運輸）
- 輸出：建議拜訪順序 + 預估所需時間，家人可一鍵發送給長者

---

## 11.5 長者端提醒優化

AI 根據長者認知狀態設定，自動調整提醒措辭的複雜度與圖示比例：

| 認知狀態 | AI 提醒措辭範例 | 設計原則 |
|---------|---------------|---------|
| 輕度認知障礙（MCI） | 王阿嬤，您快到全聯了，記得買：牛奶、雞蛋、豆腐 | 完整文字說明 + 項目清單，語氣親切 |
| 中度認知障礙 | 進全聯囉！買牛奶 🥛 雞蛋 🥚 豆腐 🫘 | 減少文字，增加圖示與 emoji，指令簡短化 |
| 一般使用者 | 到全聯了！購物清單：牛奶、雞蛋、豆腐 | 標準措辭，簡潔資訊導向 |

認知狀態由家人在後台設定，系統依據設定自動選用對應的提醒模板。Phase 3 可考慮引入 AI 自動評估機制，根據長者歷史反應速度與任務完成率動態調整。

## 11.6 技術實作方向（Phase 2）

| 功能 | 技術元件 | 免費額度 / 成本 |
|-----|---------|--------------|
| 自然語言任務解析 | Claude API（claude-haiku-4-5） | 依 token 計費，輕量解析成本極低 |
| 截圖 OCR + 解析 | Google Cloud Vision API | 每月 1,000 次免費，超出後按量計費 |
| 語音輸入 | Google Speech-to-Text v2 | 每月 60 分鐘免費（標準模型） |
| 行為分析 / 異常偵測 | 後端 Python 批次計算（Celery + Redis） | 自建邏輯，無額外 API 成本 |

**技術選型摘要：**
- LLM API（如 Claude API）：自然語言解析、任務生成、個案摘要產生
- OCR 服務（Google Cloud Vision API）：醫囑單 / 截圖文字辨識與結構化解析
- 語音轉文字（Google Speech-to-Text 或 Whisper API）：語音輸入任務建立
- 行為模式分析：後端定期批次計算（Celery 排程任務），儲存行為基線至 PostgreSQL
- 異常偵測規則引擎：可配置敏感度（高 / 中 / 低），規則以 JSON 儲存支援動態調整
- 路徑最佳化：TSP 近似演算法（Nearest Neighbor）+ Map8 路徑查詢 API
- 提醒模板系統：依 elder_mode 等級選用對應文字模板，支援 A/B 測試
