# 十、應用場景延伸 — 失智長者照護

LocaRemind 的所有功能圍繞「擷取 → 紀錄 → 提醒」三大核心流程展開。本章描述其在失智長者照護領域的延伸應用場景。此場景對產品核心架構（時間 + 地點觸發提醒）提出了更高要求，同時帶來明確的 B2B2C 商業機會（對接照護機構、醫療院所、社福單位）。

---

## 10.1 核心設計理念

### 設計轉變：從「操作者」到「接收者」

```
一般使用者模型：  使用者自行建立任務 → 自行設定時間地點 → 接收自己的提醒

失智長者照護模型：家人／社工／醫護 在後台建立並指派任務
                  ↓
                  長者手機在對的地點、對的時間 自動跳出簡單提醒
                  ↓
                  長者不需要自己設定任何東西，只需一鍵回應
```

**關鍵差異：** 長者端完全無需學習任何操作，極簡介面由照顧者在後台配置完成。

此設計理念呼應三大核心流程框架中的「被動擷取」模式：任務由他人建立，長者作為被指派的接收端。照護場景將此模式推向極致，長者甚至不需要知道任務是如何建立的，系統在對的時機自動出現。

---

## 10.2 角色設計與權限

失智長者照護場景引入四個角色，形成層層授權的信任關係：

### 1. 長者（接收端）

對應使用者：被照顧的失智長者

**核心權限：**
- 接收提醒通知（極簡介面、大字體模式）
- 一鍵完成確認（單一大按鈕，減少認知負擔）
- 緊急呼叫（一鍵通知所有已設定的家人）
- 無任務建立、設定、編輯等任何主動操作功能

### 2. 家人（主要照顧者）

對應使用者：子女、配偶等直系照顧者

**核心權限：**
- 建立、編輯、刪除長者的所有任務
- 設定長者的安全圍欄（可設多個：家、社區、醫院）
- 接收長者即時活動通知（進出圍欄、任務完成、緊急呼叫）
- 查看長者完整活動日誌與任務完成率
- 邀請社工共同管理（可授予部分任務的編輯權限）

### 3. 社工（專業服務端）

對應使用者：社區照顧服務員、日照中心人員

**核心權限：**
- 在自己的後台管理多位個案（一對多管理介面）
- 批次指派任務給多位個案（例如：所有個案今日服藥提醒）
- 查看所有個案的即時狀態與活動紀錄
- 設定定期排程任務（例如：每週二 日照中心接送）
- 需家人授權後方可管理特定長者個案

### 4. 醫療單位（醫護端）

對應使用者：診所護理師、醫院個管師

**核心權限：**
- 產生與推送醫囑任務（回診、領藥、復健、量血壓）
- 唯讀查看指定個案的任務完成率報告（不可修改個人資料）
- 設定藥局、診所、醫院地點為任務觸發地點
- 需家人明確授權後方可存取個案資料（符合 HIPAA 精神）

## 10.3 四大使用場景

### 場景一：日常領藥

| 流程步驟 | 說明 |
|---------|------|
| ① 醫護建立任務 | 護理師在醫護端後台建立「領藥提醒」，設定藥局地點、每週領藥日期、提醒內容（藥品名稱、注意事項） |
| ② 家人確認 | 家人收到任務邀請通知，確認內容正確後授權啟用，任務正式加入長者的提醒排程 |
| ③ 地點觸發提醒 | 長者（或陪同者）靠近藥局（預設圍欄半徑 100 公尺），長者手機顯示大字體提醒：「🔔 領藥時間到！請拿出健保卡」，附取藥說明 |
| ④ 完成通知 | 長者點擊「✅ 已領藥」，系統自動通知家人：「媽媽已於 14:32 完成領藥，地點：○○藥局」 |

---

### 場景二：走失預防

| 流程步驟 | 說明 |
|---------|------|
| ① 設定安全圍欄 | 家人在後台為長者設定多個「安全活動範圍」（例如：以家為中心 500 公尺、社區公園、日照中心），超出任一安全圍欄即視為異常 |
| ② 社工共同監護 | 社工可在自己的後台同時監看多位個案的即時位置狀態（顯示「在安全範圍內」或「超出範圍」），無法看到精確座標，僅看狀態 |
| ③ 超出範圍警示 | 長者離開安全圍欄超過 3 分鐘（延遲確認，避免誤報），系統同時通知家人與負責社工：「⚠️ 警示：王奶奶已超出安全範圍，最後已知位置：○○路口」 |
| ④ 長者端求助 | 長者手機顯示簡單畫面：「📍 您在安全的地方嗎？」+ 大型紅色「🆘 呼叫家人」按鈕，一鍵觸發語音通話（依序呼叫已設定的家人聯絡清單） |

---

### 場景三：日照中心接送

| 流程步驟 | 說明 |
|---------|------|
| ① 社工批次排程 | 日照中心社工在後台一次為 10 位個案設定「每週二、四 08:30 接送」排程任務，批次建立後自動指派給各長者 |
| ② 出發提醒觸發 | 接送當日 08:15，長者手機推播：「🚐 今天有日照中心！接送車 08:30 到，請準備好」；家人同步收到通知 |
| ③ 到達自動打卡 | 長者手機進入日照中心地理圍欄，系統自動完成打卡紀錄（到達時間、出發地點），家人與社工後台均可查看 |
| ④ 離開記錄 | 長者離開日照中心地理圍欄，系統記錄離開時間，通知家人：「爸爸已離開日照中心（17:05），預計 17:35 返家」 |

---

### 場景四：回診後醫囑追蹤

| 流程步驟 | 說明 |
|---------|------|
| ① 護理師產生任務清單 | 回診後，護理師在醫護端輸入醫囑：「每日早餐後量血壓（家中）」、「下週四 09:00 回診複查（本院）」、「每日 21:00 服藥（任意地點）」 |
| ② 家人授權啟用 | 家人收到授權申請，確認醫囑任務清單後一鍵啟用，任務自動綁定對應地點（家中座標、醫院座標） |
| ③ 地點與時間觸發 | 量血壓：在家且時間符合時觸發；回診：到達醫院地理圍欄時觸發；服藥：純時間觸發（21:00，不限地點） |
| ④ 完成率報告 | 護理師可在醫護端後台查看本週醫囑完成率報告（量血壓：5/7 次，服藥：7/7 次），作為下次回診的參考依據 |

## 10.4 長者端極簡 UI 設計原則

長者端介面與一般使用者介面完全分離，採用「長者模式」，由家人在後台為長者裝置開啟。

| 設計原則 | 具體實作 |
|---------|---------|
| 大字體模式 | 全介面最小字體 24pt，主要資訊 32pt 以上；圖示尺寸放大至一般的 1.5 倍；按鈕觸控區域最小 60×60pt |
| 單一動作設計 | 每則通知只有一個主要動作按鈕（例如「✅ 完成」或「📞 呼叫家人」）；避免多選項或需要判斷的操作 |
| 高對比色彩 | 背景採用米白色（#FDF6E3），文字採用深灰（#333333）；重要提示採用明顯色彩標示；符合 WCAG AA 無障礙標準 |
| 固定緊急呼叫入口 | 首頁右上角固定顯示紅色「🆘」按鈕，任何頁面均可一觸即達；長按 2 秒確認（防誤觸） |
| 無設定介面 | 長者端完全隱藏任務建立、設定、編輯、帳號管理等功能頁；所有配置均由家人端後台操作 |
| 語音輔助 | 提醒通知可設定同時播放語音（TTS，Phase 2）：「媽媽，現在是領藥時間，請按下完成按鈕」 |

## 10.5 安全圍欄警示設計

走失預防的安全圍欄與一般任務地理圍欄採用**相反邏輯**：一般任務是「進入圍欄時觸發」，安全圍欄是「**離開圍欄時警示**」。

| 設計項目 | 說明 |
|---------|------|
| 圍欄類型 | 安全圍欄（SAFETY_ZONE）：以長者慣常活動地點為圓心，家人設定半徑（建議 500 公尺至 1 公里） |
| 觸發條件 | 長者 GPS 座標連續 3 次（15 分鐘）超出安全圍欄半徑，且非已知允許地點（如醫院、日照中心） |
| 通知對象 | 家人（即時 FCM 推播）、負責社工（即時 FCM 推播）；分級：⚠️ 警告（15 分鐘未回應）→ 🆘 緊急（30 分鐘未回應） |
| 隱私保護 | 長者精確 GPS 座標不對社工顯示（僅家人可見）；社工只看到「在安全範圍內 / 超出範圍」的狀態 |
| 誤報處理 | 已登記的例行外出地點加入「允許清單」，進出不觸發警示；GPS 精度低於 50 公尺時暫緩判斷 |

## 10.6 關鍵功能需求（Phase 2 實作）

- ✦ 後台任務管理介面（Web 版，供家人 / 社工 / 醫護操作）
- ✦ 多角色權限系統（RBAC：家人 / 社工 / 醫護 / 長者端，含授權邀請流程）
- ✦ 長者端極簡 UI 模式（大字體、單一動作、固定緊急呼叫按鈕）
- ✦ 安全圍欄離開警示（區別於一般進入觸發邏輯）
- ✦ 打卡紀錄與活動日誌（進出圍欄時間戳記、任務完成歷史）
- ✦ 社工批次指派任務（一次指派給多位個案）
- ✦ 醫護端醫囑任務推送（含每日重複排程）
- ✦ 完成率報告（日 / 週 / 月維度，供醫護端唯讀查看）
- ✦ 語音提醒 TTS（長者端通知播放語音）
- ✦ 緊急呼叫依序撥打家人清單功能

## 10.7 資料模型延伸（Phase 2）

| 新增資料表 | 主要欄位說明 |
|-----------|------------|
| CareGroup（照護群組） | id、name、createdBy（家人 userId）；一位長者可屬於多個群組 |
| CareGroupMember（群組成員） | groupId、userId、role（ELDER / FAMILY / SOCIAL_WORKER / MEDICAL）、permissions（JSON）、invitedBy、joinedAt |
| SafetyZone（安全圍欄） | id、elderUserId、name、latitude、longitude、radius、allowlist（JSON，允許地點清單）、createdBy |
| ActivityLog（活動日誌） | id、elderUserId、eventType（ENTER_ZONE / EXIT_ZONE / TASK_COMPLETE / EMERGENCY_CALL / SAFETY_ALERT）、latitude、longitude、metadata（JSON）、occurredAt |
| User.elderMode（擴充欄位） | 在現有 User 表新增 elder_mode（Boolean）與 care_group_id，標示是否啟用長者極簡 UI 模式 |

## 10.8 商業模式與市場定位

| 目標客群 | 採購決策者 | 商業模式 |
|---------|-----------|---------|
| 日照中心 / 照護機構 | 機構主任 / 採購部門 | 按個案數月費訂閱（例如：每個案 $50 NTD / 月），機構統一採購 |
| 醫療院所 | 個管師 / 慢性病管理部門 | 醫囑追蹤模組授權費（依床位數 / 科室計費） |
| 社福單位 / 縣市政府 | 社工督導 / 採購單位 | 政府採購標案（配合長照 2.0 數位化政策） |
| 一般家庭（Premium） | 子女 / 主要照顧者 | 家庭照護方案訂閱（$199 NTD / 月，含長者模式 + 安全圍欄 + 活動日誌） |
